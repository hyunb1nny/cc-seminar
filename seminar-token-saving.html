03
토큰 기초

토큰 = Claude의 "글자 단위"

토큰은 단어보다 작은 단위입니다. 영어 기준 단어 하나 ≈ 1~2 토큰, 한국어는 글자 하나가 2~3 토큰.

예: "API 응답시간 분석" 토큰화
API 응 답 시 간 분 석
= 약 7토큰 (한국어는 글자당 2~3토큰 소비)
  • 입력 토큰: 내가 보내는 것 (질문 + 컨텍스트 전체)
  • 출력 토큰: Claude가 생성하는 것 (응답)
  • 비용 = (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가)

모델별 토큰 단가 (2026년 기준)

모델 입력 출력 용도
Claude Haiku 4.5 $0.80/M $4/M 단순 파싱·포맷
Claude Sonnet 4.6 $3/M $15/M 기본 작업 (권장)
Claude Opus 4.8 $15/M $75/M 복잡한 추론
1M 토큰 ≈ 약 75만 단어 ≈ 책 5권 분량. 일상 업무에서 한 달 수백만 토큰 쓰는 건 흔함.
출력 토큰이 입력보다 5배 비쌉니다. 짧게 명확하게 답하도록 요청하면 비용이 크게 줄어듭니다.

05
토큰 소비 원인

어디서 토큰이 새는가

① 긴 대화 히스토리 누적최대 원인
매 응답마다 이전 대화 전체를 다시 읽음. 50턴 대화 = 매번 50턴 분량 입력.
② 파일 전체 읽기고위험
10MB 로그 파일 Read() → 수십만 토큰 한 번에 소진.
③ MCP 도구 스키마상시 점유
등록한 MCP 서버 수 × 스키마 크기만큼 매 응답 입력에 포함.
④ 긴 Bash 출력주의
명령 결과가 수천 줄이면 전부 컨텍스트에 쌓임.
⑤ 모호한 요청 → 긴 응답출력 비용
Claude가 추측하며 길게 답 → 출력 토큰 낭비 + 히스토리 팽창.

실제 사례: 로그 분석 비교

# 나쁜 예
"app.log 분석해줘"
→ Read("app.log")
→ 200,000 토큰 소비
→
# 좋은 예
"app.log에서 ERROR
WARN만 찾아줘"
→ Grep("ERROR\|WARN", "app.log")
→ 500 토큰 소비
= 400배 차이

비용 확인 방법

# 현재 대화 비용 실시간 확인
/cost

# 출력 예시
Input:  45,231 tokens  ($0.14)
Output:  3,821 tokens  ($0.06)
Cache:  12,000 tokens  (-$0.09)
Total:                  $0.11

전략

하나씩 습관으로 만들면 비용이 눈에 띄게 줄어든다

07
절약 전략 1~4

입력 토큰 줄이기

① 작업마다 /clear

가장 효과가 큰 습관. 무관한 새 작업을 시작할 때 이전 대화를 전부 날립니다.

로그 분석 50턴 후
→ 스크립트 작성 요청
= 50턴 × 매번 읽음
→
/clear 후
스크립트 작성 요청
= 깨끗한 시작

② /compact — 이어갈 때

히스토리를 요약으로 압축. 맥락을 유지하면서 컨텍스트 크기를 줄입니다.

③ 파일은 범위 지정해서 읽기

Read("app.log")
전체: ~50,000 토큰
→
Read("app.log",
offset=0, limit=100)
일부: ~500 토큰

④ Grep으로 관련 부분만 추출

파일 전체 대신 패턴에 맞는 줄만 가져옵니다.

# 응답시간 관련 줄만 검색
Grep("response_ms", "logs/")

# 오류 패턴만 추출
Grep("ERROR|WARN|timeout",
     "logs/", recursive=True)

# Bash에서 미리 필터링
$ grep "ERROR\|WARN" app.log \
  | tail -50
Grep은 파일 내용을 컨텍스트에 모두 올리지 않고 매칭된 줄만 반환합니다. 대용량 로그에서 특히 효과적.
Bash 출력이 길면 | head -N으로 잘라주세요. Claude는 출력 전체를 컨텍스트에 저장합니다.
08
절약 전략 5~7

출력 · 구조 · MCP 최적화

⑤ 구체적인 요청 = 짧은 응답

출력 토큰은 입력보다 5배 비쌉니다. 응답을 짧게 만드는 것도 절약입니다.

"이 로그 결과 분석해줘"
→ 2,000 토큰 응답
→
"API 응답시간만 전주 대비
% 변화 한 줄 요약"
→ 50 토큰 응답

⑥ CLAUDE.md — 반복 설명 제거

매번 "우리 프로젝트는 이런 구조이고..." 설명하는 것 = 매 대화마다 수백 토큰 낭비.

CLAUDE.md는 프롬프트 캐시되므로 반복 비용이 거의 없습니다. 자세히 써도 손해가 없어요.

⑦ MCP 서버 최소화

등록된 MCP 서버의 tool schema는 쓰든 안 쓰든 항상 컨텍스트에 포함됩니다.

// MCP 서버 1개당 대략적인 비용
MongoDB MCP  ≈ 800 토큰/응답
Jira MCP     ≈ 600 토큰/응답
GitHub MCP   ≈ 1,200 토큰/응답

// 3개 등록 시 100회 응답 기준
2,600 × 100 = 260,000 토큰
= Sonnet 기준 약 $0.78 추가 비용
사용하지 않는 날에도 MCP를 켜두면 비용이 발생합니다. 프로젝트별로 필요한 것만 등록하세요.
MCP 서버를 많이 쓴다면 프로젝트별 설정 분리: .claude/settings.json에서 해당 프로젝트에 필요한 서버만 활성화.

10
모델 선택

작업에 맞는 모델 고르기

모든 작업에 Sonnet이나 Opus가 필요하지 않습니다. 작업 복잡도에 따라 모델을 선택하면 비용을 크게 줄일 수 있습니다.

Haiku 4.5 — 단순·반복 작업

  • CSV 파싱, 숫자 포맷 변환
  • 긴 로그에서 특정 패턴 추출
  • 데이터 단위 변환 (us → ms)
  • 표 형식 정리, 마크다운 변환

Sonnet 4.6 — 일반 개발·분석

  • 코드 작성 및 버그 수정
  • API 응답 결과 해석 및 리포트
  • 테스트 스크립트 작성
  • 스크립트 설계 및 구현

Opus 4.8 — 복잡한 추론만

  • 장애 원인 심층 분석 (여러 요인 교차)
  • 아키텍처 설계 결정
  • 복잡한 알고리즘 구현

Claude Code에서 모델 전환

# 설정에서 변경
/model

# 또는 settings.json
{
  "model": "claude-haiku-4-5-20251001"
}

# 사용 가능한 모델 ID
claude-haiku-4-5-20251001   # 저렴
claude-sonnet-4-6           # 기본
claude-opus-4-8             # 고성능

프롬프트 캐싱

프롬프트 캐싱 — 반복 비용 90% 절감

Claude는 컨텍스트 앞부분이 반복될 때 자동으로 캐싱합니다. 캐시된 토큰은 일반 입력 대비 90% 저렴합니다.

캐시 작동 원리
CLAUDE.md
고정 · 캐시됨
MCP 스키마
고정 · 캐시됨
대화 히스토리 (변동 · 매번 읽음)
현재 요청
캐싱은 자동입니다. CLAUDE.md를 잘 작성해두면 따로 설정 없이 자동으로 비용이 줄어듭니다.

캐싱이 적용되는 조건

  • 동일한 컨텍스트 prefix가 1,024 토큰 이상
  • 캐시 유효 시간: 5분 (5분 내 다음 요청이 오면 유지)
  • CLAUDE.md, 시스템 프롬프트, MCP 스키마가 주요 대상

실제 비용 비교 (/cost 출력)

# 캐싱 없을 때
Input:  8,400 tokens    ($0.025)
Output:   320 tokens    ($0.005)
Total:                   $0.030

# CLAUDE.md 캐싱 적용 후
Input:  1,200 tokens    ($0.004)
Cache:  7,200 tokens    ($0.001)  ← 90%↓
Output:   320 tokens    ($0.005)
Total:                   $0.010  ← 67% 절감
캐시는 5분이 지나면 만료됩니다. 자리를 비웠다가 돌아오면 첫 응답은 캐시 미스가 납니다.

로그 장애 분석

"app.log에서 14~16시
 ERROR 에러만 요약해줘"

→ Bash로 시간 필터 후 결과만 Claude에게 전달

$ awk '/14:0[0-9]|15:|16:00/
  && /ERROR/' app.log

코드 리뷰 (범위 지정)

"parse_logs.py 50~100번째
 줄의 응답시간 파싱 로직만
 검토해줘"

→ Read(offset=50, limit=50). 파일 전체 읽기 방지

"이번 달 API 로그에서

→ DB에서 필터링 → 결과만 컨텍스트 진입. 전체 fetch 금지

Summary

토큰 절약 핵심 정리

7가지 전략 체크리스트

  • ① 새 작업 시작 전 /clear
  • ② 이어갈 때 /compact로 압축
  • ③ 파일은 Read(offset, limit)으로 범위 지정
  • ④ Grep으로 필요한 줄만 추출
  • ⑤ 요청은 구체적으로 → 응답이 짧아짐
  • ⑥ CLAUDE.md로 반복 설명 제거 + 자동 캐싱
  • ⑦ MCP 서버는 실제 쓰는 것만 등록
위 7가지를 모두 적용하면 같은 작업을 5~10배 저렴하게 할 수 있습니다.

모델 선택 기준

Haiku — 파싱, 포맷, 단순 추출 (87% 절감)

Sonnet — 코드 작성, 분석, 리포트 (기본)

Opus — 복잡한 추론, 아키텍처 결정만
