03
토큰 기초

토큰 = Claude의 "글자 단위"

토큰은 단어보다 작은 단위입니다. 영어 기준 단어 하나 ≈ 1~2 토큰, 한국어는 글자 하나가 2~3 토큰.

예: "API 응답시간 분석" 토큰화
API 응 답 시 간 분 석
= 약 7토큰 (한국어는 글자당 2~3토큰 소비)
  • 입력 토큰: 내가 보내는 것 (질문 + 컨텍스트 전체)
  • 출력 토큰: Claude가 생성하는 것 (응답)
  • 비용 = (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가)

모델별 토큰 단가 (2026년 기준)

모델 입력 출력 용도
Claude Haiku 4.5 $0.80/M $4/M 단순 파싱·포맷
Claude Sonnet 4.6 $3/M $15/M 기본 작업 (권장)
Claude Opus 4.8 $15/M $75/M 복잡한 추론
1M 토큰 ≈ 약 75만 단어 ≈ 책 5권 분량. 일상 업무에서 한 달 수백만 토큰 쓰는 건 흔함.
출력 토큰이 입력보다 5배 비쌉니다. 짧게 명확하게 답하도록 요청하면 비용이 크게 줄어듭니다.
04
토큰 기초

한국어는 영어보다 토큰을 더 씁니다

Claude의 토크나이저(BPE)는 영어 중심으로 학습되었습니다. 한글 한 글자는 보통 2~3 토큰, 영어 단어 하나는 1~2 토큰.

같은 의미, 다른 토큰 수
🇺🇸 영어 — 약 6 토큰
Ana lyze API response time data
🇰🇷 한국어 — 약 14 토큰
API 응 답 시 간 데 이 터 를 분 석 해 줘

언어별 토큰 효율 비교

언어글자당 토큰같은 내용배율
영어 0.25~0.5 ~6 토큰 1×
한국어 2~3 ~14 토큰 ~2.5×
일본어·중국어 2~3 ~13 토큰 ~2×
CLAUDE.md의 규칙·지침은 영문으로 작성하면 컨텍스트 절약에 유리합니다. 매 대화마다 읽히는 파일이므로 효과가 큽니다.
대화 자체를 영어로 쓸 필요는 없습니다. 비용 차이보다 정확한 의도 전달이 더 중요합니다.

05
토큰 소비 원인

어디서 토큰이 새는가

① 긴 대화 히스토리 누적최대 원인
매 응답마다 이전 대화 전체를 다시 읽음. 50턴 대화 = 매번 50턴 분량 입력.
② 파일 전체 읽기고위험
10MB 로그 파일 Read() → 수십만 토큰 한 번에 소진.
③ MCP 도구 스키마상시 점유
등록한 MCP 서버 수 × 스키마 크기만큼 매 응답 입력에 포함.
④ 긴 Bash 출력주의
명령 결과가 수천 줄이면 전부 컨텍스트에 쌓임.
⑤ 모호한 요청 → 긴 응답출력 비용
Claude가 추측하며 길게 답 → 출력 토큰 낭비 + 히스토리 팽창.

실제 사례: 로그 분석 비교

# 나쁜 예
"app.log 분석해줘"
→ Read("app.log")
→ 200,000 토큰 소비
→
# 좋은 예
"app.log에서 ERROR
WARN만 찾아줘"
→ Grep("ERROR\|WARN", "app.log")
→ 500 토큰 소비
= 400배 차이

비용 확인 방법

# 현재 대화 비용 실시간 확인
/cost

# 출력 예시
Input:  45,231 tokens  ($0.14)
Output:  3,821 tokens  ($0.06)
Cache:  12,000 tokens  (-$0.09)
Total:                  $0.11

전략

절약 전략 7가지

하나씩 습관으로 만들면 비용이 눈에 띄게 줄어든다

07
절약 전략 1~2

입력 토큰 줄이기 — 사용자가 직접 하는 것

① 작업마다 /clear

가장 효과가 큰 습관. 무관한 새 작업을 시작할 때 이전 대화를 전부 날립니다.

로그 분석 50턴 후
→ 스크립트 작성 요청
= 50턴 × 매번 읽음
→
/clear 후
스크립트 작성 요청
= 깨끗한 시작
/clear는 히스토리를 완전 삭제. CLAUDE.md와 설정은 그대로 유지됩니다.

② /compact — 맥락은 유지하면서

히스토리를 요약본으로 압축합니다. 긴 작업을 이어갈 때 사용합니다.

대화 100턴
= 수만 토큰 누적
→
/compact 후
= 요약 ~2,000 토큰
맥락은 유지
/clear와 달리 이전 작업 결과가 요약으로 남습니다. 리팩토링처럼 긴 작업 중간에 활용하세요.
Read·Grep 같은 도구 사용 방식은 Claude(agent)가 내부적으로 결정합니다. 사용자가 직접 호출하는 것이 아닙니다.
08
절약 전략 3~5

출력 · 구조 · MCP 최적화

③ 구체적인 요청 = 짧은 응답

출력 토큰은 입력보다 5배 비쌉니다. 응답을 짧게 만드는 것도 절약입니다.

"이 로그 결과 분석해줘"
→ 2,000 토큰 응답
→
"API 응답시간만 전주 대비
% 변화 한 줄 요약"
→ 50 토큰 응답

④ CLAUDE.md — 반복 설명 제거

매번 "우리 프로젝트는 이런 구조이고..." 설명하는 것 = 매 대화마다 수백 토큰 낭비.

CLAUDE.md는 프롬프트 캐시되므로 반복 비용이 거의 없습니다. 자세히 써도 손해가 없어요.

⑤ MCP 서버 최소화

등록된 MCP 서버의 tool schema는 쓰든 안 쓰든 항상 컨텍스트에 포함됩니다.

// MCP 서버 1개당 대략적인 비용
MongoDB MCP  ≈ 800 토큰/응답
Jira MCP     ≈ 600 토큰/응답
GitHub MCP   ≈ 1,200 토큰/응답

// 3개 등록 시 100회 응답 기준
2,600 × 100 = 260,000 토큰
= Sonnet 기준 약 $0.78 추가 비용
사용하지 않는 날에도 MCP를 켜두면 비용이 발생합니다. 프로젝트별로 필요한 것만 등록하세요.
MCP 서버를 많이 쓴다면 프로젝트별 설정 분리: .claude/settings.json에서 해당 프로젝트에 필요한 서버만 활성화.

실제 벤치마크 (Scalekit, 2026)

MCP 스키마는 쓰든 안 쓰든 항상 컨텍스트에 올라옵니다. GitHub MCP 단독으로 ~1,200 토큰 상시 점유.

CLI + Skills가 유리한 경우

  • 기성 CLI 존재 (git, gh, aws, kubectl)
  • 고빈도 반복 작업 — 비용 격차가 누적됨
  • 파이프 조합이 필요한 경우 (| grep | jq)
  • 단일 개발자 환경 (공유 인증 불필요)

MCP가 필요한 경우

  • CLI 없는 SaaS — Figma, Notion, Salesforce, Workday
  • OAuth 기반 다중 사용자 인증 필요
  • 기업 감사 로그 / 거버넌스 요구사항
  • 팀 전체 표준 도구 통일이 목적
결론: 이미 CLI가 있으면 CLI 우선. CLI가 없는 SaaS에만 MCP를 씁니다.

CLAUDE.md 다이어트 — 참조는 경로로

CLAUDE.md는 매 대화마다 자동으로 컨텍스트에 올라옵니다. 내용이 클수록 항상 그만큼 소비됩니다.

CLAUDE.md (2,000줄)
# API Spec
GET /users → ...
POST /orders → ...
(전체 스펙 복붙)
= 매 대화 ~8,000 토큰
→
CLAUDE.md (50줄)
# 참조 파일
API 스펙: @docs/api.md
DB 스키마: @docs/schema.md

= 매 대화 ~200 토큰
CLAUDE.md는 색인(index)처럼 씁니다. 세부 내용은 별도 파일로 두고, Claude가 작업 중 필요할 때 Read로 가져오게 합니다.

권장 구조

# CLAUDE.md — 항상 로드됨 (작게 유지)
## 프로젝트 개요
SSD 성능 모니터링 서비스

## 참조 문서
- API 스펙: @docs/api-spec.md
- DB 스키마: @docs/schema.md
- 배포 가이드: @docs/deploy.md

## 핵심 규칙 # 여기만 직접 작성
- 테스트는 반드시 실 DB 사용
- PR은 main에 직접 push 금지
절약 효과 (예시)
CLAUDE.md 2,000줄 → 50줄
월 200회 대화 기준 ~1,560,000 토큰 절약

.claudeignore — Claude가 보지 않을 파일 지정

.gitignore처럼, Claude가 컨텍스트에 포함하지 않을 파일·디렉토리를 지정합니다. Claude Code가 자동으로 파일을 탐색할 때 적용됩니다.

왜 필요한가?

  • 대규모 파일을 실수로 읽어 컨텍스트가 터지는 것을 방지
  • node_modules, dist, *.lock 등 무관한 파일 차단
  • 민감 정보 파일 (.env, 키 파일) 보호
  • 바이너리·미디어 파일 제외
프로젝트 루트에 .claudeignore 파일을 두면 Claude Code가 자동으로 인식합니다.

예시 .claudeignore

# 빌드 산출물
node_modules/
dist/
build/
.next/
__pycache__/

# 잠금 파일 (크고 불필요)
package-lock.json
yarn.lock
poetry.lock

# 민감 정보
.env
.env.*
*.pem
secrets/

# 미디어·바이너리
*.png
*.jpg
*.mp4
*.zip

# 로그·데이터
logs/
*.log
data/*.csv

모델 선택

작업에 맞는 모델 고르기

모든 작업에 Sonnet이나 Opus가 필요하지 않습니다. 작업 복잡도에 따라 모델을 선택하면 비용을 크게 줄일 수 있습니다.

Haiku 4.5 — 단순·반복 작업

  • CSV 파싱, 숫자 포맷 변환
  • 긴 로그에서 특정 패턴 추출
  • 데이터 단위 변환 (us → ms)
  • 표 형식 정리, 마크다운 변환

Sonnet 4.6 — 일반 개발·분석

  • 코드 작성 및 버그 수정
  • API 응답 결과 해석 및 리포트
  • 테스트 스크립트 작성
  • 스크립트 설계 및 구현

Opus 4.8 — 복잡한 추론만

  • 장애 원인 심층 분석 (여러 요인 교차)
  • 아키텍처 설계 결정
  • 복잡한 알고리즘 구현

Claude Code에서 모델 전환

# 설정에서 변경
/model

# 또는 settings.json
{
  "model": "claude-haiku-4-5-20251001"
}

# 사용 가능한 모델 ID
claude-haiku-4-5-20251001   # 저렴
claude-sonnet-4-6           # 기본
claude-opus-4-8             # 고성능

프롬프트 캐싱

프롬프트 캐싱 — 반복 비용 90% 절감

Claude는 컨텍스트 앞부분이 반복될 때 자동으로 캐싱합니다. 캐시된 토큰은 일반 입력 대비 90% 저렴합니다.

캐시 작동 원리
CLAUDE.md
고정 · 캐시됨
MCP 스키마
고정 · 캐시됨
대화 히스토리 (변동 · 매번 읽음)
현재 요청
캐싱은 자동입니다. CLAUDE.md를 잘 작성해두면 따로 설정 없이 자동으로 비용이 줄어듭니다.

캐싱이 적용되는 조건

  • 동일한 컨텍스트 prefix가 1,024 토큰 이상
  • 캐시 유효 시간: 5분 (5분 내 다음 요청이 오면 유지)
  • CLAUDE.md, 시스템 프롬프트, MCP 스키마가 주요 대상

실제 비용 비교 (/cost 출력)

# 캐싱 없을 때
Input:  8,400 tokens    ($0.025)
Output:   320 tokens    ($0.005)
Total:                   $0.030

# CLAUDE.md 캐싱 적용 후
Input:  1,200 tokens    ($0.004)
Cache:  7,200 tokens    ($0.001)  ← 90%↓
Output:   320 tokens    ($0.005)
Total:                   $0.010  ← 67% 절감
캐시는 5분이 지나면 만료됩니다. 자리를 비웠다가 돌아오면 첫 응답은 캐시 미스가 납니다.
