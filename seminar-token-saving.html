Claude가 글자를 읽고 쓰는 단위

-
토큰 기초

토큰 = Claude의 "글자 단위"

토큰은 단어보다 작은 단위입니다. 영어 기준 단어 하나 ≈ 1~2 토큰, 한국어 음절 하나 ≈ 2~3 서브토큰 (아래 시각화는 음절 경계만 표시).

예: "API 응답시간 분석" 토큰화
API 응 답 시 간 분 석
= 실제로는 14~21 토큰 수준 (한글 음절당 2~3 서브토큰)
  • 입력 토큰: 내가 보내는 것 (질문 + 컨텍스트 전체)
  • 출력 토큰: Claude가 생성하는 것 (응답)
  • 비용 = (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가)

모델별 토큰 단가 (※ 가격은 변동될 수 있으니 공식 pricing 페이지에서 확인)

모델입력출력용도
Claude Haiku 4.5 $1/M $5/M 단순 파싱·포맷
Claude Sonnet 4.6 $3/M $15/M 기본 작업 (권장)
Claude Opus 4.8 $5/M $25/M 복잡한 추론
1M 토큰 ≈ 약 75만 단어 ≈ 책 5권 분량. 일상 업무에서 한 달 수백만 토큰 쓰는 건 흔함.
출력 토큰이 입력보다 5배 비쌉니다. 짧게 명확하게 답하도록 요청하면 비용이 크게 줄어듭니다.
-
토큰 기초

한국어는 영어보다 토큰을 더 씁니다

Claude의 토크나이저(BPE)는 영어 중심으로 학습되었습니다. 한글 한 글자는 보통 2~3 토큰, 영어 단어 하나는 1~2 토큰.

같은 의미, 다른 토큰 수
🇺🇸 영어 — 약 6 토큰
Ana lyze API response time data
🇰🇷 한국어 — 약 14 토큰
API 응 답 시 간 데 이 터 를 분 석 해 줘

언어별 토큰 효율 비교

언어글자당 토큰같은 내용배율
영어0.25~0.5~6 토큰1×
한국어2~3~14 토큰~2.5×
일본어·중국어2~3~13 토큰~2×
CLAUDE.md의 규칙·지침은 영문으로 작성하면 컨텍스트 절약에 유리합니다. 매 대화마다 읽히는 파일이므로 효과가 큽니다.
대화 자체를 영어로 쓸 필요는 없습니다. 비용 차이보다 정확한 의도 전달이 더 중요합니다.

원인 분석

무엇이 토큰을 많이 소비하나

줄이려면 먼저 범인을 알아야 한다

-
토큰 소비 원인

어디서 토큰이 새는가

① 긴 대화 히스토리 누적최대 원인
매 응답마다 이전 대화 전체를 다시 읽음. 50턴 대화 = 매번 50턴 분량 입력.
② 파일 전체 읽기고위험
10MB 로그 파일 Read() → 수십만 토큰 한 번에 소진.
③ MCP 도구 스키마상시 점유
등록한 MCP 서버 수 × 스키마 크기만큼 매 응답 입력에 포함.
④ 긴 Bash 출력주의
명령 결과가 수천 줄이면 전부 컨텍스트에 쌓임.

실제 사례: 로그 분석 비교

"app.log 분석해줘"
→ Read("app.log")
→ 200,000 토큰
→
"ERROR·WARN만 찾아줘"
→ Grep("ERROR|WARN")
→ 500 토큰
= 400배 차이

모델 선택

작업에 맞는 모델 고르기

모든 작업에 Sonnet이나 Opus가 필요하지 않습니다. 작업 복잡도에 따라 모델을 선택하면 비용을 크게 줄일 수 있습니다.

Haiku 4.5 — 단순·반복 작업

  • CSV 파싱, 숫자 포맷 변환
  • 긴 로그에서 특정 패턴 추출
  • 표 형식 정리, 마크다운 변환

Sonnet 4.6 — 일반 개발·분석

  • 코드 작성 및 버그 수정
  • API 응답 결과 해석 및 리포트
  • 테스트 스크립트 작성

Opus 4.8 — 복잡한 추론만

  • 장애 원인 심층 분석 (여러 요인 교차)
  • 아키텍처 설계 결정

프롬프트 캐싱

프롬프트 캐싱 — 반복 비용 90% 절감

Claude는 컨텍스트 앞부분이 반복될 때 자동으로 캐싱합니다. 캐시된 토큰은 일반 입력 대비 90% 저렴합니다.

캐시 작동 원리
CLAUDE.md
고정 · 캐시됨
MCP 스키마
고정 · 캐시됨
대화 히스토리 (변동 · 매번 읽음)
현재 요청
캐싱은 자동입니다. CLAUDE.md를 잘 작성해두면 따로 설정 없이 자동으로 비용이 줄어듭니다.

캐싱이 적용되는 조건

  • 동일한 컨텍스트 prefix가 1,024 토큰 이상
  • 캐시 유효 시간: 5분 (5분 내 다음 요청이 오면 유지)
  • CLAUDE.md, 시스템 프롬프트, MCP 스키마가 주요 대상

실제 비용 비교 (/usage 출력)

# 캐싱 없을 때
Input:  8,400 tokens    ($0.025)
Output:   320 tokens    ($0.005)
Total:                   $0.030

# CLAUDE.md 캐싱 적용 후
Input:  1,200 tokens    ($0.004)
Cache:  7,200 tokens    ($0.001)  ← 90%↓
Output:   320 tokens    ($0.005)
Total:                   $0.010  ← 67% 절감
캐시는 5분이 지나면 만료됩니다. 자리를 비웠다가 돌아오면 첫 응답은 캐시 미스가 납니다.

절약 전략 기초

후속 메시지 말고, 원본을 편집하세요

Claude가 원하는 대로 답하지 않았을 때 이런 충동이 드실 겁니다.

❌ 이렇게 하면 안 됩니다
"아니, 내 말은 이거야..."
"그게 아니라, 다시 해줘..."

후속 메시지를 보낼 때마다 Claude는 이전 대화 전체를 다시 읽습니다.

대신: Edit 버튼으로 원본 수정

✓ 원본 메시지 Edit → 수정 → 재생성
이전 대화가 쌓이지 않고 깔끔하게 유지됩니다.

절약 전략 기초

입력 토큰 줄이기

① 작업마다 /clear

가장 효과가 큰 습관. 무관한 새 작업을 시작할 때 이전 대화를 전부 날립니다.

로그 분석 50턴 후
→ 스크립트 작성 요청
= 50턴 × 매번 읽음
→
/clear 후
스크립트 작성 요청
= 깨끗한 시작
/clear는 히스토리를 완전 삭제. CLAUDE.md와 설정은 그대로 유지됩니다.

② /compact — 맥락은 유지하면서

히스토리를 요약본으로 압축합니다. 긴 작업을 이어갈 때 사용합니다.

대화 100턴
= 수만 토큰 누적
→
/compact 후
= 요약 ~2,000 토큰
맥락은 유지
/clear와 달리 이전 작업 결과가 요약으로 남습니다. 컨텍스트가 한계에 가까워지면 Claude가 자동으로 실행하기도 합니다.
-
절약 전략 기초

/handoff — 맥락을 요약해 새 세션으로 넘기기

/handoff는 현재 세션의 작업 맥락을 요약한 뒤 새 대화로 핸드오프합니다. /compact가 같은 세션 내 압축이라면, /handoff는 완전히 새로운 컨텍스트에서 재시작합니다.

커뮤니티 스킬 — mattpocock/skills에서 설치. Claude Code 내장 명령어가 아닙니다. (github.com/mattpocock/skills)
긴 대화를 그대로 이어감
= 누적 히스토리 전체 재독
= 토큰 폭증
→
/handoff
= 요약 + 새 세션 시작
= 깨끗한 컨텍스트
큰 리팩토링이나 여러 단계로 나뉜 작업에서 단계를 넘어갈 때 사용하면 효과적입니다.

/clear · /compact · /handoff 비교

명령어히스토리맥락세션
/clear삭제없음유지
/compact요약유지유지
/handoff요약유지새 세션
새 세션으로 넘어가므로 이전 대화 파일, 열린 에디터 상태는 초기화됩니다. 단계 전환 시점에 사용하세요.

④ MCP 서버 최소화

등록된 MCP 서버의 tool schema는 쓰든 안 쓰든 항상 컨텍스트에 포함됩니다.

MongoDB MCP  ≈ 800 토큰/응답
GitHub MCP   ≈ 1,200 토큰/응답
// 3개 × 100회 = 260,000 토큰 추가
사용하지 않는 날에도 MCP를 켜두면 비용이 발생합니다.

⑤ CLAUDE.md — 반복 설명 제거

매번 "우리 프로젝트는..." 설명하는 것 = 매 대화마다 수백 토큰 낭비.

CLAUDE.md는 프롬프트 캐시되어 반복 비용이 거의 없습니다. 단, 파일 자체가 클수록 첫 로드 비용은 높아지므로 꼭 필요한 내용만 담으세요.

⑥ Bash 출력 잘라내기

# 출력이 길면 앞부분만
$ pytest tests/ 2>&1 | head -50

# 오류만 필터
$ grep "ERROR\|FAIL" app.log | tail -20
Claude는 Bash 출력 전체를 컨텍스트에 저장합니다. | head -N으로 반드시 잘라주세요.

절약 전략 기초

파일 제외 설정 — Claude가 읽지 않을 경로 지정

Claude Code는 .gitignore를 자동으로 존중합니다. node_modules/, dist/ 등은 이미 파일 검색에서 제외됩니다. git에 커밋된 파일은 permissions.deny로 추가 차단합니다.

왜 필요한가?

  • vendor 코드·생성 파일을 실수로 읽어 컨텍스트 낭비 방지
  • 커밋된 대용량 파일 (SDK 번들, 스냅샷 등) 차단
  • 민감 정보 경로 보호
.gitignore에 이미 있는 경로(node_modules, dist, .env 등)는 별도 설정 불필요 — Claude Code가 자동 존중합니다.

.claude/settings.json — Read deny 규칙

// git에 커밋된 파일 중 읽지 않을 경로
{
  "permissions": {
    "deny": [
      "Read(./**/dist/**)",
      "Read(./**/build/**)",
      "Read(./**/*.generated.*)",
      "Read(./vendor/**)"
    ]
  }
}

CLI vs MCP — 토큰 실측 비교

실측 벤치마크 (Scalekit, 2026 — 동일 작업 비교)

MCP 스키마는 쓰든 안 쓰든 항상 컨텍스트에 올라옵니다.

CLI가 유리한 경우

  • 기성 CLI 존재 (git, gh, aws, kubectl)
  • 고빈도 반복 작업 — 비용 격차가 누적됨
  • 파이프 조합 (| grep | jq) 필요
  • 터미널에서 직접 실행 가능 — 유저와 Agent가 같은 결과를 확인할 수 있어 디버깅이 쉬움

MCP가 필요한 경우

  • CLI 없는 SaaS — Figma, Notion, Salesforce
  • OAuth 기반 다중 사용자 인증 필요
  • 기업 감사 로그 / 거버넌스
결론: 이미 CLI가 있으면 CLI 우선. CLI가 없는 SaaS에만 MCP를 씁니다.

Subagent — 독립 컨텍스트로 격리해 절약

Subagent는 별도 컨텍스트 윈도우에서 실행됩니다. 메인 세션의 긴 히스토리 없이 작업을 처리하고 결과만 반환합니다.

메인 세션 (50,000 토큰 누적)
Agent("로그 파일 분석해줘")
↳ 새 컨텍스트 (0 토큰에서 시작)
↳ 분석 완료 후 결과만 반환
= 메인 세션은 결과 텍스트만 추가
  • 메인 세션에 불필요한 중간 결과가 쌓이지 않음
  • 여러 Subagent를 병렬 실행해 작업 분담
  • 실패해도 메인 컨텍스트가 오염되지 않음

적합한 작업 유형

대용량 파일 분석

로그, CSV — 메인 세션에 올리지 않고 Subagent가 요약만 반환

병렬 코드 리뷰

여러 PR을 동시에 리뷰 — 각각 독립 컨텍스트, 결과만 취합

탐색 작업 격리

관련 파일 탐색 — 탐색 과정을 메인 컨텍스트에 남기지 않음

Graphify — 프로젝트를 Knowledge Graph로 만들어 Grep 제거

CLI 도구 + Claude Code 스킬. /graphify . 한 번으로 프로젝트 전체를 지식 그래프로 변환합니다. 이후 파일을 일일이 읽지 않고 필요한 노드만 쿼리합니다.

기존 방식 vs Graphify
❌ 기존 — 파일 전체 읽기
Grep → Read(file) → Read(file2) → ... = 수만 토큰
✓ Graphify — 그래프 쿼리
query_graph("인증 관련 코드") → 관련 노드만 = 수백 토큰
한 번 생성한 그래프는 여러 대화에서 재사용됩니다. 생성 비용은 1회, 절약은 매번.

설치 및 사용 (CLI + Skill)

# 1. CLI 설치
uv tool install graphifyy

# 2. Claude Code 스킬 등록
graphify install

# 3. 프로젝트 그래프 생성
/graphify .

# 4. Claude가 스킬 명령어로 쿼리
query_graph("auth")
get_neighbors("UserService")
  • God nodes — 핵심 허브 개념 자동 식별
  • 36개 언어 지원 (tree-sitter 기반, API 호출 없음)
  • PreToolUse hook — 파일 읽기 전 그래프 먼저 조회
🔗 github.com/safishamsi/graphify
